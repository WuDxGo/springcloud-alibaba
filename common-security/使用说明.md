# OAuth2 资源服务器安全自动配置使用说明

## 概述

`common-security` 模块提供了 OAuth2 资源服务器的**自动配置**，新增微服务模块**无需手动配置 SecurityConfig**，实现"无感"安全保护。

---

## 模块职责划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| `common` | Feign 拦截器、工具类等通用组件 | OAuth2 Resource Server（用于获取 JWT） |
| `common-security` | OAuth2 资源服务器安全自动配置 | Spring Security + OAuth2 Resource Server |

---

## 新增模块使用步骤

### 步骤 1：在 pom.xml 中引入依赖

```xml
<!-- 公共组件（Feign 拦截器等） -->
<dependency>
    <groupId>com.xiao</groupId>
    <artifactId>common</artifactId>
    <version>${project.version}</version>
</dependency>

<!-- OAuth2 资源服务器安全自动配置 -->
<dependency>
    <groupId>com.xiao</groupId>
    <artifactId>common-security</artifactId>
    <version>${project.version}</version>
</dependency>
```

### 步骤 2：在 application.yml 中配置 OAuth2

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          # JWK 密钥集 URI（指向 SSO 认证中心）
          jwk-set-uri: http://localhost:8080/oauth2/jwks
```

### 步骤 3（可选）：自定义安全配置

如果需要**自定义放行路径**或**修改 JWT 转换器**，可通过配置文件实现：

```yaml
common:
  security:
    # 是否启用自动安全配置（默认 true）
    enabled: true
    # 放行路径（无需认证）
    permit-all-patterns:
      - /actuator/**
      - /api/public/**
      - /health
    # 权限前缀（默认 ROLE_）
    authority-prefix: "ROLE_"
    # 权限在 JWT 中的字段名（默认 authorities）
    authorities-claim-name: "authorities"
```

**完成！无需编写 SecurityConfig 类。**

---

## 自动配置生效条件

| 条件 | 说明 |
|------|------|
| 类路径存在 `EnableWebSecurity` | 确保引入了 Spring Security |
| 类路径存在 `OAuth2ResourceServerProperties` | 确保引入了 OAuth2 资源服务器 |
| 容器中不存在自定义 `SecurityFilterChain` Bean | 如果自定义了配置类，自动配置会失效，使用自定义的 |
| `common.security.enabled=true` | 可通过配置禁用自动配置 |

---

## 完全自定义场景

如果默认配置不满足需求，可以**自定义 SecurityConfig**：

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        // 自定义安全配置
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/custom/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2.jwt(...));
        
        return http.build();
    }
}
```

**注意**：一旦自定义 `SecurityFilterChain` Bean，自动配置会自动失效（`@ConditionalOnMissingBean` 生效）。

## 示例：新增 user-service 模块

### 1. 创建模块并添加依赖

```xml
<!-- user-service/pom.xml -->
<dependencies>
    <!-- 公共组件 -->
    <dependency>
        <groupId>com.xiao</groupId>
        <artifactId>common</artifactId>
        <version>${project.version}</version>
    </dependency>
    
    <!-- 安全自动配置 -->
    <dependency>
        <groupId>com.xiao</groupId>
        <artifactId>common-security</artifactId>
        <version>${project.version}</version>
    </dependency>
    
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    </dependency>
</dependencies>
```

### 2. 配置 application.yml

```yaml
server:
  port: 8084

spring:
  application:
    name: user-service
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://localhost:8080/oauth2/jwks
```

### 3. 启动服务

无需任何额外配置，安全保护自动生效！

---

## 配置属性说明

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `common.security.enabled` | boolean | true | 是否启用自动安全配置 |
| `common.security.permit-all-patterns` | String[] | `/actuator/**` | 放行路径列表 |
| `common.security.authority-prefix` | String | `ROLE_` | 权限前缀 |
| `common.security.authorities-claim-name` | String | `authorities` | JWT 中权限字段名 |

---

## 架构优势

| 优势 | 说明 |
|------|------|
| **零配置** | 新增模块只需引入 `common-security` 依赖 |
| **可覆盖** | 支持自定义 SecurityConfig 覆盖默认配置 |
| **可配置** | 通过 YAML 配置文件即可调整安全策略 |
| **职责分离** | `common` 负责通用组件，`common-security` 负责安全配置 |

---

## 迁移说明（针对已有模块）

如果你的模块已有 `SecurityConfig.java`：

1. **保留自定义配置**：如果配置与默认不同，保留即可，自动配置不会生效
2. **删除冗余配置**：如果配置与默认相同，可直接删除 `SecurityConfig.java` 并清理 pom.xml 中的冗余依赖

清理后的 pom.xml 应只保留：
```xml
<dependency>
    <groupId>com.xiao</groupId>
    <artifactId>common</artifactId>
    <version>${project.version}</version>
</dependency>
<dependency>
    <groupId>com.xiao</groupId>
    <artifactId>common-security</artifactId>
    <version>${project.version}</version>
</dependency>
```
