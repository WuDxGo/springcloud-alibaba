### 2.2 请求流程

```
步骤 1: 用户访问 Gateway → http://localhost:8081/order/callCf99
        ↓
步骤 2: Gateway 发现未登录 → 重定向到 OAuth2-SSO 登录页
        ↓
步骤 3: 用户在 SSO 页面登录 → 输入用户名密码
        ↓
步骤 4: SSO 验证成功 → 生成授权码 → 回调 Gateway
        ↓
步骤 5: Gateway 用授权码换取 Token → 创建会话
        ↓
步骤 6: Gateway 转发请求到 Order Service → 携带 JWT Token
        ↓
步骤 7: Order Service 验证 JWT → 处理业务逻辑 → 返回结果
        ↓
步骤 8: Gateway 返回响应给用户
```

### 4.3 Spring Security 6 配置

#### 知识点

- **SecurityFilterChain**：安全过滤器链，定义请求处理流程
- **AuthenticationManager**：认证管理器
- **PasswordEncoder**：密码编码器
- **@EnableMethodSecurity**：启用方法级安全注解

#### Spring Security 6 新特性

```java
// 旧版本写法 (5.x)
@Override
protected void configure(HttpSecurity http) throws Exception {
    http.authorizeRequests()
        .antMatchers("/public/**").permitAll()
        .anyRequest().authenticated();
}

// 新版本写法 (6.x) - Lambda DSL
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(authorize -> authorize
            .requestMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
        );
    return http.build();
}
```

#### 项目实践

**Gateway 安全配置 (WebFlux)**：

```java
@Configuration
@EnableWebFluxSecurity
public class SecurityConfig {

    @Bean
    public SecurityWebFilterChain securityWebFilterChain(ServerHttpSecurity http) {
        http
            // 禁用 CSRF (WebFlux OAuth2 登录需要)
            .csrf(ServerHttpSecurity.CsrfSpec::disable)
            
            // 配置授权规则
            .authorizeExchange(exchanges -> exchanges
                .pathMatchers("/oauth2/**", "/login/**", "/logout", "/error").permitAll()
                .anyExchange().authenticated()
            )
            
            // 启用 OAuth2 登录
            .oauth2Login(oauth2 -> oauth2
                .authorizedClientRepository(authorizedClientRepository())
            )
            
            // 启用 OAuth2 资源服务器 (JWT 验证)
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt
                    .jwtAuthenticationConverter(jwtAuthenticationConverter())
                )
            )
            
            // 配置登出
            .logout(logout -> logout.logoutUrl("/logout"))
            
            // 配置异常处理 - 未认证时重定向到 OAuth2 登录
            .exceptionHandling(exceptions -> exceptions
                .authenticationEntryPoint((exchange, e) -> {
                    String redirectUrl = "/oauth2/authorization/gateway-client";
                    exchange.getResponse().setStatusCode(HttpStatus.FOUND);
                    exchange.getResponse().getHeaders().set("Location", redirectUrl);
                    return exchange.getResponse().setComplete();
                })
            );
        
        return http.build();
    }
}
```

### 4.5 JWT (JSON Web Token)

#### JWT 结构

```
Header.Payload.Signature

Header:
{
  "alg": "RS256",
  "typ": "JWT"
}

Payload:
{
  "sub": "admin",
  "iat": 1680000000,
  "exp": 1680007200,
  "authorities": ["ROLE_ADMIN", "ROLE_USER"],
  "scope": ["read", "write"]
}

Signature:
RSA256(
  base64(header) + "." + base64(payload),
  私钥
)
```

#### JWT 验证流程

```java
// 1. Gateway 从 OAuth2-SSO 获取 JWT
OAuth2AccessToken accessToken = authorizedClient.getAccessToken();

// 2. Gateway 将 JWT 传递给后端服务
template.header("Authorization", "Bearer " + tokenValue);

// 3. 后端服务验证 JWT 签名
// 通过 JWK Set URI 获取公钥验证签名
// 验证 issuer、exp、iat 等声明

// 4. 从 JWT 中提取用户信息
Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
if (authentication instanceof JwtAuthenticationToken jwtToken) {
    String username = jwtToken.getToken().getSubject();
    Collection<String> authorities = jwtToken.getToken().getClaim("authorities");
}
```

#### JWT 权限转换器

```java
@Bean
public JwtAuthenticationConverter jwtAuthenticationConverter() {
    // 配置权限转换器
    JwtGrantedAuthoritiesConverter grantedAuthoritiesConverter = 
        new JwtGrantedAuthoritiesConverter();
    grantedAuthoritiesConverter.setAuthorityPrefix("ROLE_");
    grantedAuthoritiesConverter.setAuthoritiesClaimName("authorities");

    JwtAuthenticationConverter jwtAuthenticationConverter = 
        new JwtAuthenticationConverter();
    jwtAuthenticationConverter.setJwtGrantedAuthoritiesConverter(grantedAuthoritiesConverter);
    return jwtAuthenticationConverter;
}
```

**Token 传递过滤器**：

```java
@Component
public class OAuth2TokenRelayFilter implements GlobalFilter, Ordered {

    @Autowired
    private OAuth2AuthorizedClientRepository authorizedClientRepository;

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        // 从会话中获取 OAuth2 Token
        return exchange.getPrincipal()
            .cast(OAuth2AuthenticationToken.class)
            .flatMap(oauth2Token -> {
                String registrationId = oauth2Token.getAuthorizedClientRegistrationId();
                
                // 加载已授权的客户端
                return authorizedClientRepository.loadAuthorizedClient(
                        registrationId, oauth2Token, exchange)
                    .flatMap(authorizedClient -> {
                        OAuth2AccessToken accessToken = authorizedClient.getAccessToken();
                        
                        if (accessToken != null) {
                            String tokenValue = accessToken.getTokenValue();
                            
                            // 将 Token 添加到请求头传递给后端服务
                            ServerWebExchange mutatedExchange = exchange.mutate()
                                .request(builder -> builder
                                    .header(HttpHeaders.AUTHORIZATION, "Bearer " + tokenValue))
                                .build();
                            
                            return chain.filter(mutatedExchange);
                        }
                        return chain.filter(exchange);
                    });
            })
            .onErrorResume(e -> chain.filter(exchange));
    }

    @Override
    public int getOrder() {
        return -100; // 高优先级
    }
}
```

### 5.2 核心配置类

#### AuthorizationServerConfig - OAuth2 服务器配置

```java
@Configuration
@EnableWebSecurity
public class AuthorizationServerConfig {

    @Bean
    public SecurityFilterChain authorizationServerSecurityFilterChain(HttpSecurity http)
            throws Exception {
        http
            // 配置 OAuth2 授权服务器的安全策略
            .securityMatcher(OAuth2AuthorizationServerConfiguration.authorizationServerEndpointRequest())
            .authorizeHttpRequests(authorize -> authorize
                .anyRequest().authenticated()
            )
            // 启用 OAuth2 授权服务器
            .with(new OAuth2AuthorizationServerConfiguration(), customizer -> {})
            // 配置表单登录
            .oauth2ResourceServer(oauth2 -> oauth2.jwt(Customizer.withDefaults()));
        
        return http.build();
    }

    @Bean
    public RegisteredClientRepository registeredClientRepository(DataSource dataSource) {
        // 基于 JDBC 存储 OAuth2 客户端
        return new JdbcRegisteredClientRepository(dataSource);
    }

    @Bean
    public JWKSource<SecurityContext> jwkSource() {
        // 生成 RSA 密钥对用于 JWT 签名
        KeyPair keyPair = generateRsaKey();
        RSAPublicKey publicKey = (RSAPublicKey) keyPair.getPublic();
        RSAPrivateKey privateKey = (RSAPrivateKey) keyPair.getPrivate();
        
        RSAKey rsaKey = new RSAKey.Builder(publicKey)
            .privateKey(privateKey)
            .keyID(UUID.randomUUID().toString())
            .build();
        
        JWKSet jwkSet = new JWKSet(rsaKey);
        return (jwkSelector, context) -> jwkSelector.select(jwkSet);
    }

    private KeyPair generateRsaKey() {
        KeyPairGenerator keyPairGenerator;
        try {
            keyPairGenerator = KeyPairGenerator.getInstance("RSA");
            keyPairGenerator.initialize(2048);
            return keyPairGenerator.generateKeyPair();
        } catch (Exception e) {
            throw new IllegalStateException("Failed to generate RSA key pair", e);
        }
    }
}
```

#### DefaultSecurityConfig - 默认安全配置

```java
@Configuration
@EnableWebSecurity
public class DefaultSecurityConfig {

    @Bean
    public SecurityFilterChain defaultSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            // 配置 CORS
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            // 禁用 CSRF
            .csrf(csrf -> csrf.disable())
            // 配置授权规则
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/api/login", "/oauth2/**", "/error", "/static/**").permitAll()
                .anyRequest().authenticated()
            )
            // 配置表单登录
            .formLogin(form -> form
                .loginPage("/login")
                .permitAll()
            )
            // 启用 OAuth2 资源服务器 (JWT 验证)
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(jwtAuthenticationConverter()))
            );
        
        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        // 允许的跨域来源
        configuration.setAllowedOrigins(Arrays.asList(
            "http://localhost:3000",
            "http://localhost:8081",
            "null"  // 允许本地文件访问
        ));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowCredentials(true);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }
}
```

## 10. 常见问题

### 10.1 环境搭建问题

#### Q1: 数据库连接失败

**问题**: `Communications link failure`

**解决方法**:
1. 确认 MySQL 已启动
2. 检查端口是否正确 (3307)
3. 检查用户名密码
4. 确认数据库已创建

#### Q2: Nacos 连接失败

**问题**: `Connect to Nacos server failed`

**解决方法**:
1. 确认 Nacos 已启动 (端口 8848)
2. 检查 namespace 是否正确
3. 检查用户名密码

### 10.2 OAuth2 集成问题

#### Q1: 登录失败 401

**可能原因**:
- client_id 或 client_secret 错误
- redirect_uri 不匹配
- 授权模式配置错误

**解决方法**:
```bash
# 检查 Basic Auth 格式
echo -n "gateway-client:gateway-secret" | base64
# 确保 Authorization 头为：Basic <base64 编码>
```

#### Q2: Token 验证失败

**问题**: 后端服务无法验证 JWT

**解决方法**:
1. 确保配置了正确的 JWK URI
```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          jwk-set-uri: http://localhost:8080/oauth2/jwks
```
2. 检查 OAuth2-SSO 服务是否正常运行
3. 检查网络连通性

#### Q3: 重定向 URI 不匹配

**问题**: `Redirect URI mismatch`

**解决方法**:
确保请求中的 redirect_uri 与客户端配置**完全一致**：
- 协议 (http/https)
- 域名/IP
- 端口
- 路径

### 10.3 网关问题

#### Q1: 网关无法转发请求

**问题**: 访问网关返回 404

**解决方法**:
1. 检查路由配置
2. 检查服务是否注册到 Nacos
3. 检查 Predicate 配置

#### Q2: Token 无法传递到后端

**问题**: 后端服务收不到 Token

**解决方法**:
1. 检查 OAuth2TokenRelayFilter 是否生效
2. 检查过滤器优先级
3. 查看网关日志

### 10.4 前端问题

#### Q1: 跨域错误

**问题**: `Access-Control-Allow-Origin`

**解决方法**:
1. 后端配置 CORS
2. 前端配置代理 (vite.config.ts)

#### Q2: Token 过期处理

**问题**: Token 过期后如何处理

**解决方法**:
1. 使用 Refresh Token 刷新
2. 跳转到登录页重新登录
3. Axios 拦截器统一处理
